## Что это

KER — это личный ИИ-ассистент, который ты держишь у себя. Общаешься с ним в
Telegram или в десктопном приложении, а под капотом работает настоящий движок:
он помнит ваши разговоры, вызывает инструменты, чтобы реально что-то сделать,
говорит и слушает, выполняет задачи по расписанию и умеет подключаться к умному
дому.

Две вещи отличают его от «ещё одной обёртки над чатом»:

- **Он твой.** Никакое имя не вшито намертво. Назови его KER, назови как
  угодно — каждый может дать своему ассистенту собственное имя, и он будет на
  него отзываться (и на второй псевдоним тоже, если захочешь). Свой ключ, свой
  сервер, свои данные.
- **Это цельный продукт, а не скрипт.** Аккуратный Telegram-бот на кнопках
  вместо команд, десктопный «Command Deck» с живым дашбордом, аккаунты и
  тарифы, автообновления — всё, что реально нужно, если ты хочешь отдать это
  другим людям (или продавать).

Всё работает локально. Простые вещи (время, калькулятор, диагностика) работают
даже без API-ключа.

> **⚡ Бесплатная и полная версия.** Здесь, на GitHub, — только **бесплатное
> ядро**, его хватает, чтобы запустить собственного ассистента. **Полная версия**
> (все премиум-функции и повышенные лимиты) доступна **только через Telegram-бота**,
> по подписке. Запусти бота: [@jar_v1_s](https://t.me/jar_v1_s).

## Что он умеет

**Говорить и помнить.** Общение текстом или голосом. У него настоящая память —
он помнит ваши разговоры после перезапуска и тихо выделяет из них устойчивые
факты о тебе, чтобы вспомнить их позже. Вставленные пароли, токены и номера карт
вырезаются ещё до того, как что-либо сохранится.

**Реально действовать.** Модель не просто отвечает — она может вызывать
инструменты: искать в интернете, читать и писать файлы, выполнять команды в
терминале, управлять компьютером, смотреть погоду, общаться с умным домом.
Опасные возможности по умолчанию выключены и включаются по одной.

**Подключать новые инструменты (MCP).** KER говорит на Model Context Protocol,
так что инструменты любого MCP-сервера становятся родными навыками ассистента.
Укажи конфиг или добавь сервер прямо из дашборда.

**Работать самостоятельно.** Скажи «каждый день в 9:00 делай сводку» или
«каждые 3 часа проверяй почту» — он поставит задачу в расписание, выполнит её,
пришлёт результат и сам перепланирует следующий запуск.

**Понимать твой язык.** Весь интерфейс и ответы на русском, английском и
узбекском. Пришли голосовое — он расшифрует, ответит и может проговорить ответ,
на том языке, на котором ты говорил. Голос можно запустить полностью бесплатно
и офлайн.

**Свой «мозг» на выбор.** Работает с Anthropic (Claude), OpenAI (GPT),
OpenRouter или локальной моделью (Ollama, LM Studio, vLLM, llama.cpp) — если
берёшь локальную, облачный ключ вообще не нужен. Сам делает повторы и
переключение между провайдерами, а роутер отправляет простые вопросы на быструю
модель, а сложные — на сильную.

## Сделай его своим (white-label)

Именно то, чего обычно не дают другие ассистенты.

- **Дай имя.** По умолчанию — **KER**, но любой пользователь может переименовать
  своего ассистента прямо в настройках бота: новое имя появится в меню, в
  ответах и на десктопном дашборде. Оператор может задать имя по умолчанию для
  всей установки.
- **Дополнительные имена.** Задай `ASSISTANT_ALIASES` — и он будет отзываться на
  несколько имён (удобно как личное wake-слово). По умолчанию пусто, чтобы
  копия, которую ты отдаёшь кому-то, оставалась «чистой».
- **Твой ключ, твой сервер, твои данные.** Пользователь может даже подключить
  собственный API-ключ (BYOK). Ничего никуда не отправляется втихую.

```env
ASSISTANT_NAME=KER
ASSISTANT_ALIASES=Jarvis   # опционально — отзывается на оба
```

## Быстрый старт

**Понадобится:** Python 3.10+ и API-ключ (или локальная модель).

```bash
git clone https://github.com/nomadnairi/K.E.R.git
cd K.E.R

python -m venv venv
source venv/bin/activate         # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env             # добавь ANTHROPIC_API_KEY (или OPENAI_API_KEY),
                                 # либо укажи локальную модель
python -m jarvis
```

Любишь `make`? `make install`, `make run`, `make test`, `make lint`.

## Общение в Telegram

Самый удобный способ. Это бот на кнопках — никаких команд запоминать не нужно —
с отдельной сессией на каждого, так что тебя он помнит независимо от других.

```bash
pip install aiogram

# .env:
#   TELEGRAM_BOT_TOKEN=...   (у @BotFather)
#   ANTHROPIC_API_KEY=...    (или OPENAI_API_KEY, или локальная модель)

python -m jarvis.interfaces.telegram_bot     # или: jarvis-bot
```

Внутри — настройки, спрятанные в аккуратные вложенные меню: язык, имя
ассистента, модель ИИ, память, голос, интеграции и другое. Пришли голосовое —
ответит голосом. Есть тарифы (Free / Plus / Pro) с лимитами по плану, если
захочешь запустить это как продукт.

## Десктоп — «Command Deck»

`jarvis-desktop` — это настоящее десктопное приложение (PySide6, собирается в
Windows `.exe`) с живым веб-дашбордом внутри: домашний экран с анимированным
реактором и телеметрией системы, чат с историей, каталог моделей, MCP-серверы и
настройки — всё подключено к настоящему движку через встроенный локальный API и
обновляется в реальном времени.

Владелец запускает его локально с полным контролем над своим ПК; остальные
могут войти по логину/паролю или по **коду входа из Telegram**, который выдаёт
бот, и получить ограниченную версию. Приложение умеет само проверять и ставить
обновления.

Загрузки (установщик для Windows + портативная сборка) — на странице
[**Releases**](https://github.com/nomadnairi/K.E.R/releases). Как собирать
`.exe` / `.apk` — в [docs/CLIENTS.md](docs/CLIENTS.md).

## Запуск на сервере

Бот и HTTP/WebSocket-API на VPS через Docker:

```bash
cp .env.example .env      # впиши свои ключи
docker compose up -d --build
```

Бот работает через long-polling (входящий порт не нужен), API слушает `:8000`,
оба используют постоянные тома `data/` и `logs/`. systemd-юнит, пример nginx +
TLS и чек-лист безопасности — в [docs/DEPLOY.md](docs/DEPLOY.md).

Другие приложения могут общаться с тем же движком по API: `GET /health`,
`POST /chat` и WebSocket `/ws/{session}` для стриминга. Перед публичным доступом
задай `API_KEY`.

## Под капотом

Python 3.10+, всё асинхронно. Провайдер-независимый LLM-клиент с повторами и
фолбэком, система навыков/инструментов с быстрым детерминированным путём, память
на SQLite с семантическим поиском, шина событий pub/sub, слой безопасности с
гейтингом возможностей, FastAPI + WebSocket и Telegram-бот на aiogram.
Конфигурация типизирована (pydantic-settings). Тесты — pytest, линт — ruff на
каждом коммите.

Полная картина и статус компонентов — в [VISION.md](VISION.md) и
[ROADMAP.md](ROADMAP.md), архитектура — в
[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
